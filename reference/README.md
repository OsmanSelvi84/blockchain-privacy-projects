# Reference Implementation

While working on this project, I reviewed an open-source CoinShuffle implementation to better understand how transaction mixing works.

Reference repository:

https://github.com/atong01/coinshuffle

The reference project includes participant communication, transaction handling, and output shuffling mechanisms.

My project is a simplified version created for educational purposes. Instead of implementing a full blockchain network, it focuses on the main idea of CoinShuffle:

* collecting participant outputs,
* shuffling output addresses,
* reducing transaction traceability,
* demonstrating transaction unlinkability.

The reference repository was used only to understand the CoinShuffle concept and overall system design.
