# Original Implementation

This folder contains my original implementation of the Ring Signature Anonymous Messaging project.

## Overview

The purpose of this application is to demonstrate how ring signatures can be used for anonymous message signing and verification.

While working on the project, I used the `pyring` reference implementation to better understand how one-time ring signatures work in practice. After testing the reference project, I developed a separate menu-based application that provides a simpler interface for signing and verifying messages.

The application also demonstrates message integrity. If a signed message is modified, the signature can no longer be verified successfully because the message hash changes.

## Features

- Anonymous message signing
- Ring signature verification
- SHA-256 message hashing
- Message integrity verification
- Ring member information display
- Dynamic ring size calculation
- Signature metadata generation
- Terminal-based user interface
- Signer anonymity preservation
- Solidity smart contract demonstration

## How to Run

Run the application:

```bash
python3 main.py
```

## Menu

```text
1. Sign anonymous message
2. Verify signature
3. Show ring information
4. Exit
```

## Key Generation

The original implementation uses key pairs generated by the reference implementation.

Generate keys using:

```bash
cd ../reference_implementation/pyring
ring-keygen
```

Generated public keys (`*.pub`) are used as ring members and are displayed through the Ring Information menu.

## Example

Example message:

```text
Blockchain privacy is important
```

Example verification result:

```text
Valid ring signature.
The message was signed by one of the ring members.
Exact signer identity remains anonymous.
```

If the message is modified after signing, verification will fail.

## Implementation Details

The implementation uses the same `pyring` one-time ring signature algorithm as the reference implementation.

During signing:

- the message is converted into bytes,
- ring member public keys are loaded,
- the signer private key is loaded,
- the ring signature is generated,
- the signature is stored as `signature.pem`.

During verification:

- the signature is loaded from `signature.pem`,
- the message is loaded from metadata,
- the signature is verified,
- the verifier can confirm that the signature belongs to one of the ring members,
- the exact signer cannot be identified.

## Metadata

The application creates a metadata file named:

```text
signature_metadata.json
```

The metadata contains:

- original message,
- message hash,
- ring size,
- ring member list,
- verification result,
- privacy information.

The signer identity is not stored in any output file.

## Ring Information

The Ring Information menu displays:

- current ring size,
- public keys that belong to the ring.

Ring size is determined by the number of public keys available in the ring.

## Solidity Integration

The project includes a simple Solidity smart contract:

```text
contracts/AnonymousMessageRegistry.sol
```

The contract demonstrates how verification-related information could be stored on-chain, including:

- message hash,
- verification status.

The signer identity is intentionally not stored on-chain.
