# Final Evaluation Guidelines

## 1. General Requirements

Each student must complete the following requirements.

---

## A. Reference Implementation

Students must find and study at least one reference implementation, open-source repository, research prototype, or interface related to their assigned topic.

The reference implementation must:

- be shared with the instructor,
- include a runnable version,
- and be executable for comparison purposes.

Students must clearly provide:

- repository link,
- setup instructions,
- dependency information,
- and execution steps.

---

## B. Original Implementation

Students must develop their own original implementation from scratch.

Requirements:

- Direct copying is strictly prohibited.
- Reference projects may only be used for learning and comparison purposes.
- The final implementation must reflect the student’s own understanding and development effort.

Students must upload their final project to their assigned GitHub branch.

All project-related files must be committed to the assigned branch, including:

- source code,
- README documentation,
- configuration files,
- dependencies,
- scripts,
- and demo/test materials.

---

## C. README Documentation (Mandatory)

Each project repository must include a detailed README.md file.

The README must allow the instructor to:

- clone/download the repository,
- switch to the correct student branch,
- install dependencies,
- build the project,
- run the implementation successfully,
- test the project,
- and reproduce the demo environment.

The README should include:

- project description,
- branch information,
- installation steps,
- required software/tools,
- dependency installation,
- build commands,
- run commands,
- sample inputs/outputs,
- testing instructions,
- screenshots or demo explanations (optional but recommended).

If the instructor cannot successfully run the project by following the README instructions, points may be deducted.

---

# 2. GitHub Submission Rules

Students must submit their work through their assigned GitHub branch.

Requirements:

- Do not submit to another student’s branch.
- Do not overwrite shared project files unrelated to your branch.
- Commit history should clearly reflect the student’s own work and development process.

Students must upload before the presentation date:

- the final version of the original project,
- README documentation,
- dependency/configuration files,
- test/demo materials,
- and the reference implementation information.

Both:
- the student implementation,
- and the reference implementation

must be runnable independently by the instructor.

# 4. Evaluation Criteria (100 Points Total)

## Part A — Reference vs. Original Implementation Comparison (50 Points)

Students must compare:

- the outputs of the reference implementation,
- with the outputs of their own implementation.

Both implementations must be runnable during evaluation.

### Evaluation Method

The instructor will provide 5 different test inputs.

For each test case:

1. The reference implementation will be executed.
2. The student’s implementation will be executed.
3. Outputs and behaviors will be compared.

If the outputs match correctly:

- the student receives 10 points.

Total:
- 5 × 10 = 50 points

### Important Notes

- Functional equivalence is required.
- Minor UI differences are acceptable.
- Correctness of logic and privacy functionality is the primary evaluation criterion.

---

## Part B — Oral Examination and Technical Understanding (50 Points)

This section evaluates the student’s theoretical understanding and implementation knowledge.

### A. Theoretical Questions (20 Points)

Students will answer:
- 2 theoretical questions related to their assigned topic.

Possible topics include:

- cryptographic foundations,
- privacy-preserving mechanisms,
- blockchain integration,
- security assumptions,
- protocol design,
- attack resistance,
- scalability and limitations.

Evaluation criteria:

- conceptual accuracy,
- technical understanding,
- clarity of explanation.

---

### B. Code Understanding and Revision Tasks (30 Points)

Students must demonstrate complete understanding of their own implementation.

The instructor may:

- ask questions about specific parts of the code,
- request explanation of functions or algorithms,
- remove a portion of the code and ask the student to reconstruct or revise it,
- ask the student to debug or improve a selected section.

Students will answer:
- 3 implementation/code-related questions or tasks.

Evaluation criteria:

- correctness,
- coding logic,
- software architecture understanding,
- debugging ability,
- implementation knowledge.

---

# 5. Academic Integrity Rules

- Plagiarism is strictly prohibited.
- Copy-paste implementations without understanding will receive failing grades.
- Students must be able to explain every major component of their project.
- External repositories, papers, libraries, or tutorials used during development must be properly referenced.

---

# 6. Expected Deliverables

Before the presentation day, students should have:

- a working original implementation,
- a runnable reference implementation,
- a complete GitHub branch submission,
- a detailed README file,
- test/demo scenarios,
- presentation slides (optional but recommended),
- and sufficient technical understanding to answer all questions.

---

The purpose of this evaluation is not only to produce working software, but also to demonstrate:

- understanding of blockchain privacy technologies,
- cryptographic reasoning,
- secure implementation practices,
- reproducible software engineering workflows,
- and independent development capability.
